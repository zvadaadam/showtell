# Showtell

**Agents don't just tell. They show.**

Showtell is an open-source motion engine for videos made by agents, for humans.
An agent writes a small spec—or a richer bundle with programmable browser
HyperFrames—and Showtell renders a polished, narrated MP4.

It is designed for agent-made explainers, PR walkthroughs, product demos,
release recaps, lessons, reports, and visual stories. Code and diff scenes point
to real files and Git refs, so Showtell reads the live source instead of asking
an agent to paste it into a video.

## Install

Showtell supports Apple Silicon macOS and glibc-based Linux on x64 or ARM64.

```bash
brew install ffmpeg
npm install --global showtell
showtell version
```

On Ubuntu or Debian:

```bash
sudo apt-get update
sudo apt-get install -y ffmpeg espeak-ng
npm install --global showtell
showtell version
```

You can also run it without installing globally:

```bash
npx showtell help
```

The npm package selects one self-contained Bun-compiled binary and its pinned
Chromium runtime for your platform. You do not need Bun or a separate browser
install to run it.

## Let an agent use it

Install the bundled Showtell skill for your coding agent, then ask naturally:

> Make me a showtell of this PR.

> Turn this architecture into a two-minute narrated video.

> Create a launch video from the changes on this branch.

The skill teaches the agent to gather source material, author the spec, review
stills or exact-timestamp motion filmstrips, and render the final video:

```bash
showtell skill install
```

For Codex, choose its skill directory explicitly:

```bash
showtell skill install --dir ~/.codex/skills
```

## Make your first showtell

Create `hello.spec.json`:

```json
{
  "meta": {
    "title": "What changed",
    "aspectRatios": ["16:9"],
    "repo": { "path": "." }
  },
  "scenes": [
    {
      "kind": "title",
      "content": { "heading": "What changed" },
      "narration": "Here is the short version of this change.",
      "duration": "auto"
    },
    {
      "kind": "code",
      "content": {
        "file": "README.md",
        "lineStart": 1,
        "lineEnd": 18,
        "focus": [1]
      },
      "narration": "The project introduces itself right here.",
      "duration": "auto"
    }
  ]
}
```

Validate it, review fast stills, then render:

```bash
showtell validate hello.spec.json
showtell render hello.spec.json --frames-only --out .showtell/frames
showtell render hello.spec.json --out .showtell/out
```

Showtell emits structured JSON, including output paths and actionable `hint`
fields when something needs fixing.

## Simple specs and full bundles

For a quick video, use one declarative `spec.json` with these scene kinds:

- `title`
- `code`
- `diff`
- `talking-points`
- `chart`
- `screencap`

Showtell compiles the designed simple-spec scenes to trusted internal browser
HyperFrames, so quick authoring and crafted bundles share one Chromium visual
runtime. `screencap` is recorded media composited into that production pipeline,
not a separate design renderer.

For a richer video, use a Showtell bundle:

```text
my-video.showtell/
  spec.json
  hyperframes/*.html
  assets/**
  compiled-plan.json  # generated by Showtell
```

Every designed bundle v3 scene is a deterministic HTML/CSS/GSAP browser
HyperFrame (`visual.kind: "web"`). Use renderer-provided web components for
straightforward code, diff, and chart scenes or compose a custom browser world.
Recorded sessions use the first-class `visual.kind: "screencap"` media arm.
They let an agent choreograph code, data, maps, screenshots, captions,
diagrams, and animation against narration without calculating frames or
writing ffmpeg commands.

Browser HyperFrames contain executable JavaScript. Showtell constrains their
runtime with static policy checks, a content security policy, blocked network
requests, and deterministic API guards, but this is not a hostile-code
sandbox. Review a bundle's `hyperframes/` before rendering it, just as you
would review a script, and do not render bundles from untrusted sources.

```bash
showtell bundle components
showtell bundle themes
showtell bundle validate my-video.showtell
showtell bundle inspect my-video.showtell
showtell bundle workshop my-video.showtell --out .showtell/workshop
showtell bundle review my-video.showtell --out .showtell/review --aspect 16:9,9:16
showtell bundle render my-video.showtell --out .showtell/out --aspect 16:9,9:16
```

`bundle workshop` is a fast held-state layout gallery. It captures one
representative exact frame per narration line for judging hierarchy, overflow,
and responsive composition. `bundle review` is the motion proof; it samples
multiple exact timestamps per narration line and writes filmstrips for judging
motion, caption restraint, and responsive adaptation before the final render.
These commands compile automatically; run `bundle compile` directly only when
you want to inspect the generated `compiled-plan.json` without rendering.

See the [bundle v3 authoring guide](docs/bundle-v3.md) and the
[browser-runtime example](examples/bundle-v3).

## How it stays in sync

Narration uses `"duration": "auto"`. Showtell first synthesizes and measures the
spoken audio, then lays out the visual timeline against that measured duration.
Identical resolved inputs on the same pinned runtime and platform produce the
same frames, and narration is cached per line.

Local narration works without an API key through macOS `say` or Linux
`espeak-ng`. OpenAI, Replicate, and ElevenLabs voices are also supported through
environment credentials.

## Useful commands

```bash
showtell help
showtell schema
showtell validate <spec.json>
showtell render <spec.json> --out <dir>
showtell preview <spec.json>
showtell bundle inspect <bundle-dir>
showtell bundle workshop <bundle-dir>
showtell bundle review <bundle-dir>
showtell bundle render <bundle-dir> --out <dir>
showtell capture analyze --id <capture-id>
```

Every command is non-interactive, flag-driven, and agent-readable. Run
`showtell help` for examples and the complete command surface.

## Requirements and current limits

- Apple Silicon macOS, or glibc-based Linux on x64/ARM64
- `ffmpeg`
- `espeak-ng` on Linux when using the default local narration provider
- Git 2.24 or newer for repository references
- Screen Recording permission only when capturing on macOS

Rendering existing screenshots and screencap files works on Linux. Recording a
new screen session remains macOS-only in `0.1.x`.
